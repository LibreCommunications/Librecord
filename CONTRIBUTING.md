# Contributing to Librecord

Thanks for your interest in contributing. This is a solo project, so please be patient with response times.

## Getting started

1. **Open an issue first** — describe what you want to change and why. This avoids wasted effort on something that might not fit the project direction.
2. **Fork the repo** and create a branch from `master`.
3. **Make your changes** — keep PRs focused on a single thing.
4. **Test your changes** — run `dotnet test` for backend, `pnpm lint` and `pnpm build` for frontend.
5. **Submit a pull request** against `master`.

See [docs/development.md](docs/development.md) for instructions on running Librecord locally.

## Guidelines

- One PR per change. Don't bundle unrelated fixes.
- Describe what changed and why in the PR description.
- Follow the existing code style. Don't reformat code you didn't change.
- Make sure existing tests pass before submitting.
